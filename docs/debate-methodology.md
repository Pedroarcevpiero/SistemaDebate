# Metodología del debate por rondas

## Filosofía

El sistema no busca ganar una discusión retórica: busca **producir conocimiento**
sometiendo una tesis a presión adversarial estructurada. Dos posiciones fuertes y un
árbitro imparcial generan, al final, un veredicto trazable y una fuente reutilizable.

## Roles

- **thesis-framer**: convierte una afirmación en una pregunta debatible con dos posiciones
  simétricas y criterios objetivos. No decide.
- **proponent-debater**: defiende la tesis con el mejor caso posible.
- **opponent-debater**: ataca la tesis con la mejor objeción posible.
- **debate-arbiter**: enmarca reglas, controla el foco, puntúa y declara ganador.

## Qué es una ronda

Una ronda = una intervención de cada posición. Con N rondas hay 2N intervenciones
principales más el veredicto final.

```
Ronda 1: A abre        → B responde
Ronda 2: A replica     → B contrarreplica
...
Ronda N: A cierra       → B cierra
Veredicto del árbitro
```

`A` y `B` se asignan según quién inicia (proponente u oponente).

## Contexto acumulado

Cada intervención recibe todo el historial previo: tesis reformulada, criterios,
intervención anterior del rival, rondas previas, evidencia usada y preguntas pendientes.
Esto fuerza respuestas directas y evita repeticiones.

## Evidencia

La evidencia web se usa cuando el tema lo exige (datos actuales, técnicos, financieros,
legales, regulatorios o de mercado; afirmaciones verificables del rival; hechos externos).
Toda evidencia externa debe citar la fuente y explicar su efecto en el argumento. **No se
inventan fuentes.**

## Veredicto

El árbitro puntúa cada criterio de 0 a 10 para ambas posiciones, declara ganador
(o empate técnico) y documenta qué evidencia podría cambiar el resultado.
